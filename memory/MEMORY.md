## 🔑 5가지 절대 규칙

### 1. 하네스 5단계 작업 프로세스 (절대 규칙)

> **코드 변경이 필요한 모든 요청에 5단계 프로세스를 반드시 진행한다.**
> **회의 없이 코드 작성 시작 금지.**

```
STEP 1. Triage      (T19 Harness Librarian)
STEP 2. 회의        (병렬 호출 — R1 CTO 포함)
STEP 3. 구현        (도메인 에이전트 + 공유 역할)
STEP 4. 다중 검증   (자체 → 리뷰 → QA → CTO)
STEP 5. 보고·회고   (T18 자가 진화)
```

자세히: [.claude/HARNESS.md](../.claude/HARNESS.md)

### 2. CTO 게이트 (절대 규칙)

> **CTO 사인오프 없이 기술 결정 금지.**
> 일단 코딩하고 나중에 통보 → 금지.

게이트 발동 트리거 (8가지): 의존성·DB·외부 API·보안·인프라·핵심 알고리즘·디자인 토큰·아키텍처

자세히: [.claude/skills/cto-review-gate.md](../.claude/skills/cto-review-gate.md)

### 3. 다중 검증 4단계 (절대 규칙)

> **"한 번 만들어 굴러가니 끝"이라는 평가는 금지.**
> 4단계 모두 통과해야 사인오프.

```
① 자체 검증 (구현자)
② 코드 리뷰 (T13 — 1차/2차/3차)
③ QA 테스트 (T12 — 골든 패스 + 엣지 + 환각 검출 + 회귀)
④ CTO 사인오프 (R1 — 아키텍처·보안·성능·기술 부채)
```

어느 단계 실패 시 → STEP 3 구현으로 복귀.

자세히: [.claude/HARNESS.md](../.claude/HARNESS.md) §5

### 4. 감사 로그 (절대 규칙)

> **모든 데이터 변경 API에 `writeAuditLog()` 동시 구현.**
> "나중에 추가"는 금지.

대상: POST / PUT / PATCH / DELETE 모두

자세히: [.claude/skills/audit-log-pattern.md](../.claude/skills/audit-log-pattern.md)

### 5. 자가 진화 (절대 규칙)

> **세션 종료 시 T18이 회고 + 메모리 저장.**
> 회고 없이 종료하면 다음 세션이 같은 실수 반복.

```
KPT 회고 → 학습 추출 → 분류 → 저장 → 다음 사이클 적용
```

자세히: [.claude/skills/self-evolution-loop.md](../.claude/skills/self-evolution-loop.md)

## ⚠️ 주의사항

1. **회의 없이 코드 작성 금지** — 5단계 프로세스 준수
2. **CTO 게이트 우회 금지** — "PM이 OK했으니" 같은 우회 금지
3. **단일 검증으로 OK 금지** — 4단계 모두 통과해야 함
4. **감사 로그 누락 금지** — 변경 API와 동시 구현
5. **회고 없이 세션 종료 금지** — T18 호출 강제
6. **근거 부족 시 패널 숨김** — 후기 0건 → 노출 X (T3 규칙)
7. **AI는 결정 안 함, 옵션만 제시** — Live Replan, 사용자 결정권 (T5 규칙)
8. **검증되지 않은 정보 노출 금지** — 5단계 검증 후만 (T4 규칙)
9. **알레르기 매칭은 100% 확정만** — 추측 금지 (S-08 규칙)
10. **위치 데이터 프라이버시** — 동의·최소·기기 내 처리 (T16 규칙)
