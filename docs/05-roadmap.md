# 05. Roadmap

## MVP 기능 우선순위 — Tier 시스템

### 🟢 Tier 1 — MVP 출시 필수

이게 없으면 우리가 아님.

1. **AI 일정 생성 + 추천 근거 패널** (M1)
2. **환각 차단 검증 레이어** (5단계)
3. **동선 최적화 + 지도 뷰**
4. **Live Replan 엔진** (M3)
5. **메뉴판 카메라 번역** (M4)

### 🟡 Tier 2 — 출시 후 1~2개월

6. 한국어 후기 통합 요약 (네이버 블로그 LLM 인덱스)
7. 예약 딥링크 (Klook/KKday/Agoda 가격 비교)
8. 카카오톡 일행 공유 + 동시 편집

### 🔴 Tier 3 — 의도적으로 미루기 (트랩)

만들고 싶어지지만 MVP에서 절대 만들면 안 됨:

- ❌ 자체 결제·예약 시스템 (PG 수수료, CS 부담)
- ❌ 항공권 검색 엔진 (GDS 연동 수십억 비용)
- ❌ 커뮤니티/배낭톡 (콜드스타트 지옥)
- ❌ 여행 가계부·환전 (토스/트래블월렛이 이미 잘 함)
- ❌ 오프라인 모드 (캐싱·동기화 복잡도)

## 화면 LEVEL 분류

총 17개 화면 중 정성도를 다르게 적용:

### 🔴 LEVEL 1 — 정성껏 (인터랙티브 프로토타입)

매직 모먼트 + 첫인상. 한 화면당 1~2시간.

- ✅ **온보딩** (4단계) — `app/onboarding/`
- ⏳ **일정 생성 중** — `app/itinerary/creating/`
- ⏳ **일정 상세 (M1: 추천 근거)** — `app/itinerary/[id]/item/[itemId]/`
- ⏳ **여행 중 홈 (M2)** — `app/travel/[id]/`
- ⏳ **Live Replan (M3)** — 모달 또는 별도 페이지
- ⏳ **카메라 번역 (M4)** — `app/translate/`
- ⏳ **공개 일정 갤러리** — 콘텐츠 마켓 정체성

### 🟡 LEVEL 2 — 표준 (스타일 시스템으로 빠르게)

한 화면당 30분.

- 홈 (여행 전)
- 일정 전체
- 일정 편집
- 항목 검색·추가
- 일정 미리보기
- 일행 초대
- 공유 참여
- 투표·동의
- 자동 정산
- 여행 기록

### 🟢 LEVEL 3 — 명세만 (텍스트 + 와이어 스케치)

디자인 거의 안 함. 개발자 알아서.

- 로그인
- 알림센터
- 내 일정 공개
- 마이페이지
- 설정

## 시드 데이터 전략 — 푸꾸옥 800곳

### Phase 0 (출시 전 2주)

- 푸꾸옥 핵심 800곳 사전 크롤링 → 자체 DB 캐싱 (사이클 1엔 24곳 큐레이션 시드)
- 네이버 블로그 푸꾸옥 글 8천 개 → LLM 사전 요약
- 검증된 메타데이터 + 환불 정책 정적 DB
- 결과: API 호출 비용 80% 절감

### Phase 1 (출시 후 1개월)

- 캐시 히트 → DB 즉시 응답
- 캐시 미스 → API 호출 + DB 추가
- Google Places 비용 최적화

### Phase 2 (1만 MAU 후)

- 사용자 일정 패턴 자체가 자산
- "이 일정 평균 4.7점" 우리만의 데이터
- 외부 API 의존도 점점 ↓
