# 시드니 사이트 패턴 인벤토리 (2026-04-29 추출)

> **출처**: https://sydney-trip-production.up.railway.app
> **목적**: 시드니 사이트가 채우고 있는 "여행 동반자 기본기"를 추출해 TravelDiary v2에 반영.

---

## 1. 사이트 형태 (한 줄)

**한국인 자유여행자 1명을 위해 손으로 정밀하게 짜인 단일 페이지 가이드 + 5개 보조 도구**.
우리 MVP가 "AI 자동 생성"이라면, 시드니는 "사람 기획자의 정밀 큐레이션".

---

## 2. 메인 페이지 — Day 1~10 일정 + 4개 고정 박스

### 2-1. 일정 항목 데이터 필드 (시드니가 관리하는 정밀도)

```
시각 · 장소명 · 카테고리(파인다이닝/투어/관광/이동)
예약 정보 (확정/대기/취소)
가격 (AU$ + 원화 환산)
인원 (성인 × 2)
연락처 (전화 — tel: 링크)
지도 (Google Maps 좌표 링크)
드레스코드 (캐주얼/스마트캐주얼/정장)
집합 장소 (피어/주소)
준비물 (개별 항목)
특별 요청 (창가 자리·테라스)
취소 정책 (24h 이내 AU$100 등)
결제 상태 (사전결제/홀딩피/현장결제)
바우처 번호 / 영수증 링크
외부 예약 플랫폼 (SevenRooms, OpenTable, Klook, 마이리얼트립)
공식 웹사이트 링크
```

→ 우리 `ItineraryItem.bookingStatus`는 `{ provider, bookingId, refundable }` 3필드. **시드니는 15+ 필드**. 큰 갭.

### 2-2. 응급 정보 박스 (하단 고정)

```
🚨 호주 통합 응급 — 000
🇰🇷 한국 영사관 — +61 2 9210 0234
📞 한국 외교부 24시간 콜센터 — +82 2 3210 0404
🏨 호텔 (체크인 시)
🚕 택시 13 22 27
🚇 대중교통 긴급
```

→ **항상 화면 하단에 고정**. 클릭 시 `tel:` 다이얼.

### 2-3. 현지 정보 박스

```
💳 결제·환전: 현지 카드 사용 우선, ATM 수수료
🚇 교통: Opal 카드, 페리 노선, 우버 평균 요금
⚡ 전압: Type I 어댑터
🌐 유심·esim: Telstra/Optus
```

### 2-4. 자주 가는 곳 우버 바로가기

```
호텔 → 오페라하우스 (Uber 직링크)
호텔 → 본다이비치 (Uber 직링크)
...
```

→ 좌표 기반 deeplink (`uber://?action=setPickup&dropoff[latitude]=...`)

### 2-5. 카드 "보여주기" — 상황별 영문 카드

```
🏨 호텔 — 체크인/룸서비스/연장 5문장
🍽️ 레스토랑 — 예약/주문/계산/알레르기 6문장
☕ 카페 — 5문장
🚕 교통 — 우버 호출/택시/길묻기 5문장
📸 관광 — 사진 부탁/입장 5문장
🛍️ 쇼핑 — 가격/세일/택스 5문장
🆘 긴급 — 도움/병원/경찰 5문장
```

→ 점원/택시기사에게 화면 보여주는 용도. **대화형이 아닌 카드형 정적 데이터**.

## 3. /book — 예약 관리

```
카테고리: Fine Dining 최우선 / 캐주얼 / 기타
필드: 예약자명 · 인원 · 예약번호 · 날짜·시간 · 가격 · 메뉴 · 드레스코드 · 집합 · 준비물 · 특별 요청
상태: ✅ 확정 / ⭐ 최우선 / (옵션)
환불·취소: 정책 텍스트 + 결제 상태
바우처/영수증: 외부 링크 (#receipt-aria 형태)
외부: 🎟️ SevenRooms · 🌐 ariasydney.com.au · 📞 tel: · 🗺️ Google Maps
알림: "투어 전날 안내 메시지 마이리얼트립 앱으로 발송"
```

❌ 검색·필터·자동 알림 없음 (정적 정보만)

---

## 4. /cost — 비용 관리

```
이중 통화: KRW + AUD
환율: 1 AUD = X KRW 표시
결제 상태: 결제완료 / 예약완료 / 예정
직접 추가: 자유 항목명 (카테고리 강제 없음)
자동 저장: "이 기기에 자동 저장"
동기화 키: 공유 키 입력으로 다른 기기/일행과 데이터 공유  ← 협업의 단순한 첫 단계
총 예상 합계
```

❌ 1인당 분배·예산 비교·영수증 첨부·차트 없음

## 5. /checklist — D-Day 타임라인 체크리스트

```
D-30: 비자(ETA) · 항공권 · 숙소 · 투어 예약
D-14: 보험 · eSIM · 환전 · 오프라인 지도
D-7: 짐싸기 · 서류 인쇄 · 온라인 체크인
D-1: 최종 점검

카테고리:
- 서류 (기내 지참): 여권, ETA 이메일, 항공권, 호텔/투어 바우처
- 의류 (계절·기온): 패딩, 정장, 운동화
- 전자기기: Type I 어댑터, 보조배터리
- 금지 반입: 육류·유제품·생과일·채소 (호주 검역 명시)
- 신고 가능: 개별포장 과자, 진공포장 반찬
- 특별: Vivid 드론쇼 일정, 투어 일정
```

→ **국가별 검역·전압·비자 등 사실 정보까지 포함**. 우리 MVP에 0%.

## 7. /vivid — 큐레이션 가이드 (도시 시그니처 이벤트)

```
사진 명소 13곳 (필수 ⭐ + 추가)
6.5km Light Walk 동선 — 구간별 거리·시간·하이라이트
시간대별 운영 (18:00~23:00)
스마트폰 야간 촬영 팁 (나이트모드, 격자, HDR off)
혼잡 피하기 (요일·시간대 가이드)
준비물 (옷·신발·배터리)
Night-by-Night 루트 — Day 일정과 연결
외부 링크: 구글맵 13개·우버·Vivid 공식앱
```

→ **도시별 시그니처 경험을 위한 별도 가이드**. M1 근거 패널이 데이터라면, vivid는 **스토리텔링**.

## 8. 외부 링크 표준 (시드니 패턴)

| 아이콘 | 의미 | 형태 |
|-------|------|------|
| 🗺️ | Google Maps | `maps.google.com/maps?q=...` |
| 🚕 | Uber deeplink | `uber://?dropoff[lat]=...` |
| 📞 | 전화 다이얼 | `tel:+...` |
| 📧 | 이메일 | `mailto:` |
| 🎟️ | 예약 플랫폼 | SevenRooms·OpenTable·Klook |
| 🌐 | 공식 사이트 | https |
| 💬 | 메신저 | 마이리얼트립 등 |
| 🎫 | 바우처 | 내부 #anchor |

→ **우리 ItineraryItem 카드에 이 8개 아이콘이 노출되어야 시드니 정밀도 도달.**

## 9. 우리 MVP 대비 핵심 갭 (한 줄 요약)

```
1. 데이터 모델 정밀도        ← 시드니 15+ 필드 vs 우리 3필드
2. 응급/실용/검역/전압/통신   ← 시드니 풍부 vs 우리 0
3. D-Day 체크리스트           ← 시드니 4단계 타임라인 vs 우리 0
4. 비용 관리·동기화 키        ← 시드니 단순하지만 동작 vs 우리 0
5. 외부 링크 8종 표준 아이콘   ← 시드니 통합 vs 우리 부분
6. 상황별 영어 카드           ← 시드니 7카테고리·40+ 문장 vs 우리 0
7. 도시 시그니처 큐레이션     ← 시드니 vivid 1건 vs 우리 0
8. 자동 저장 + 동기화 키 협업 ← 시드니 단순하지만 동작 vs 우리 0
```

이 8가지가 사이클 6 견적 기획의 베이스라인.
