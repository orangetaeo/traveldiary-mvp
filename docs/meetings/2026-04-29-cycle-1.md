### A2. 푸꾸옥 시드 큐레이션 (T8 + T3)
24곳:
- **food (8)**: 즈엉동 야시장(시푸드), 더 캐비, 윤스키친, 럭셔리 시푸드, 코코넛 마을, 옌식당, 로컬 분짜집, 호텔 조식 데모
- **spot (8)**: 빈원더스, 빈펄 사파리, 호국사(쭈아 호꾸옥), 솔라이라 케이블카(혼톰), 사오비치, 디아동(별이 보이는 절벽), 함닌 어촌마을, 즈엉동 야시장(거리)
- **shopping (4)**: 빈콤 플라자, K-Market, 야시장 잡화, 그랜드월드 베네치아
- **rest (4)**: 사오비치 일몰, 그랜드월드 곤돌라, 짱오 폭포, 호텔 풀

### A3. 시연 일정 (DAG)
3박 4일, Day 0~3 (dayIndex 0-base):
- **Day 1**: 호텔 체크인 → 즈엉동 야시장(food) → 사오비치 야경(rest)
- **Day 2**: 빈원더스(spot) → 점심 → 빈펄 사파리(spot) → 더 캐비 디너(food)
- **Day 3**: 사오비치(spot) → 솔라이라 케이블카(spot) → 호국사(spot) → 디아동 일몰(rest)
- **Day 4**: 함닌 어촌(spot) → 출발

각 ItineraryItem에 Evidence 정적 작성 (한국어 후기 수·긍정율·출처).

### A4. Prisma 스키마 (T14 + R1)
- S-09의 v1 스키마 채택. 단, **사이클 1 첫 마이그레이션 이름** = `0001_init_pqc`.
- DAG 영속화: 인접 리스트 (`ItineraryDependency`).
- Evidence는 `Json` 컬럼으로 직렬화 저장.
- AuditLog 모델 포함. 변경 API 없으므로 사이클 1엔 비어 있음.
- DATABASE_URL 비어 있으면 `lib/prisma.ts`는 throw 하지 않고 **dummy 처리** → 화면은 시드를 직접 사용.

### A5. M1 화면 명세 (T17)
- `app/itinerary/creating/page.tsx`: 4단계 진행률 (취향분석→1000곳 검토→동선 최적화→환각 차단). 12초 시뮬레이션 → 자동 navigation.
- `app/itinerary/[id]/page.tsx`: Day 탭 + 일정 카드 리스트. 카드는 시간·이름·카테고리 배지·근거 미리보기 1줄.
- `app/itinerary/[id]/item/[itemId]/page.tsx`: 일정 상세 + EvidencePanel(기본 접힘).
- 디자인 토큰만 사용. 하드코딩 hex/px 금지. 카테고리별 배지 색상:
  - food → amber, spot → purple, shopping → ink, rest → success.
