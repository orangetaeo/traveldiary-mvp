# 03. Style System

화면 4개에서 자연스럽게 추출된 디자인 언어. 새 화면 만들 때 항상 이 룰을 따르세요.

코드: `lib/design-tokens.ts`, `tailwind.config.ts`에 동기화됨.

---

## 1. 컬러 — 5색 + 회색

### 룰
1. 색은 의미를 동반해야 한다. 예쁘다고 색을 추가하지 않음.
2. 빨강은 진짜 위험에만 (알레르기, 환불 불가). 그 외 사용 금지.
3. 보라 ↔ 코랄로 모드 구분. (여행 전 vs 여행 중)
4. 한 화면에 색 3개 이내. 메인 1개 + 보조 1개 + 회색.

### 의미 매핑

| 색상 | 의미 | 사용처 |
|------|------|--------|
| 🟣 보라 (`purple`) | 계획·정보·AI 추천 | 추천 근거 패널, 예약 완료, 카테고리 "관광" |
| 🟠 코랄 (`accent`) | 진행·시간·여행 중 | 여행 중 헤더, 펄스 점, 현재 일정 |
| 🟡 앰버 (`amber`) | 주의·지연·사회적 증거 | 웨이팅 시간, 지연, "한국인 BEST" |
| 🔴 빨강 (`danger`) | 안전·알레르기 | 알레르기 함유, 환불 불가 |
| 🟢 초록 (`success`) | 안전·완료·여행 후 | 예약 완료, "안전" 옵션 |
| ⚪ 회색 (`ink-soft`) | 중성·기본·구조 | 그 외 모든 곳 |

## 2. 타이포그래피 — 5단계만

| 크기 | 용도 | 예시 |
|------|------|------|
| 22px / Medium | 화면 헤더 | "13:42", "푸꾸옥 3박 4일" |
| 16~18px / Medium | 카드 메인 제목 | "스시 다이" |
| 14px / Regular | 항목 제목, 본문 | 일반 텍스트 |
| 12px / Regular | 보조 정보 | "네이버 후기 92%", 부제 |
| 11px / Regular | 라벨, 캡션 | 시간, 거리 |

**룰**: 한 화면에 4단계 이내. 5단계 다 쓰면 위계 무너짐.

## 3. 컴포넌트 인벤토리

화면 4개에서 반복적으로 등장한 패턴 6가지. 코드는 `components/ui/`.

### Card (3가지 변형)

```tsx
<Card variant="plain">     // 회색 배경, 보조 정보
<Card variant="raised">    // 흰색 + 회색 테두리, 주요 단위
<Card variant="featured">  // 흰색 + 2px 보라 테두리, 한 화면 1개만
```

### Badge (5가지 톤)

```tsx
<Badge tone="info">추천</Badge>           // 보라
<Badge tone="amber">웨이팅 30분</Badge>    // 앰버
<Badge tone="danger">⚠ 새우 함유</Badge>   // 빨강
<Badge tone="success">예약 완료</Badge>     // 초록
<Badge tone="neutral">90분</Badge>         // 회색
```

룰:
- 2~8자 짧은 라벨
- 항상 카드 우상단 또는 항목 옆
- 본문 안에 박지 않음

### FilterChip (토글)

```tsx
<FilterChip active={true} onClick={...}>맛집 위주</FilterChip>
<FilterChip variant="danger" active={true}>새우 알레르기</FilterChip>
```

룰:
- 한 줄 가로 스크롤
- 알레르기 칩은 항상 가장 왼쪽 + variant="danger"

### Button (3단계 위계)

```tsx
<Button variant="secondary">대안 보기</Button>     // 흰색 — 보조
<Button variant="primary">예약 비교</Button>        // 보라 — 진행
<Button variant="decisive">적용</Button>            // 검정 — 결정. 한 화면 1개만.
```

### EvidencePanel (우리 정체성)

```tsx
<EvidencePanel
  label="왜 이걸 골랐나"
  reasons={["네이버 후기 92% 긍정", "도보 8분", "현지인 비율 60%"]}
/>
```

룰:
- 항상 접힘 상태에서 시작
- 라벨은 "왜 이걸 골랐나" / "변경 영향" / "검증 정보"

### ImpactDisplay (변경 영향 — 추후 추가)

```tsx
<ImpactDisplay items={[
  { key: "디너 예약", value: "19:00 그대로", tone: "positive" },
  { key: "시부야 도착", value: "30분 지연", tone: "negative" },
  { key: "추가 비용", value: "¥600", tone: "neutral" },
]} />
```

Live Replan과 일정 교체에 사용.

## 4. 여백·간격

밀도 제어가 핵심. **여백 = 정보 우선순위**.

| 간격 | 용도 |
|------|------|
| 4px | 같은 의미 단위 안 (제목과 부제) |
| 8px | 같은 그룹 안 (버튼 사이) |
| 12px | 카드 안 섹션 사이 |
| 16px | 카드 사이 |
| 24px | 큰 섹션 사이 |

**카드 패딩**: `12px 16px` (위아래 12, 좌우 16).
**룰**: 의문 들 때는 더 넓게. 모바일은 빽빽함보다 여유로움이 좋음.

## 5. 인터랙션 패턴 — 6가지

### 5-1. 탭 → 펼침
근거 패널, 대안 보기, "전체 보기"
- 신호: 작은 ⌃ / › 화살표
- 피드백: 0.25초 부드러운 높이 변화

### 5-2. 탭 → 즉시 교체
대안 항목 탭 → 메인 카드 변경 + 상단 배너 알림
- 확인 다이얼로그 없음. 되돌릴 수 있다는 신뢰 위에서.

### 5-3. 탭 → 모달 (하단 시트)
예약 비교, 상세 정보
- 모달은 **항상 하단에서 올라옴**. 한 손 사용 고려.

### 5-4. 토글 (필터 칩, 모드 전환)
- 켜고 끄기. 시각적으로 명확한 on/off 상태.

### 5-5. FAB (떠다니는 버튼)
- 여행 중 모드에만 등장. 화면 우하단 고정.
- 핵심: 카메라 번역(보라). 보조: 주변 검색(흰색).
- **여행 전엔 절대 노출 금지** — 모드 차이를 드러내는 신호.

### 5-6. 자동 알림 배너
- 일정 변경, 자동 갱신 등 사용자가 안 한 변화.
- 상단 4초 표시 후 자동 사라짐.
- 색: 초록(긍정 변화) 또는 앰버(주의 필요).

## 6. 화면 구조 템플릿

새 화면 만들 때 이 템플릿으로 시작:

```
┌────────────────────────┐
│  헤더 (12~14px 메타)   │  ← 5초 안에 답: "이 화면 뭐야?"
│  ──────────────────── │
│  요약/통계 (선택)      │  ← 8초 안에 답: "지금 상황이 어때?"
│  ──────────────────── │
│  메인 콘텐츠           │  ← 30초 안에 답: "내가 뭘 알아야 해?"
│  (카드들 / 리스트)     │
│  ──────────────────── │
│  CTA 영역 (선택)       │  ← 항상 답: "이제 뭐 해?"
└────────────────────────┘
```
