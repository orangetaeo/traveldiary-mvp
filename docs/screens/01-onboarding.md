# Screen 01. Onboarding

**LEVEL**: 1 (정성껏)
**상태**: ✅ 구현 완료 (`app/onboarding/page.tsx`)
**다음 화면**: `02-itinerary-creating.md`

## 답해야 할 단 하나의 질문

> **"30초 안에 이 앱이 나에게 줄 가치를 보여주는가?"**

## 5가지 핵심 결정

### 1. 회원가입 vs. 익명 시작 → **익명 시작**

트리플이 회원가입 강제로 진입 장벽 ↑. 우리는 가치 본 뒤에 자연스럽게 유도.
회원가입 시점은 "일정 저장 시" — 일정을 다 만든 후.

### 2. 입력 단계 수 → **최대 4단계**

4단계 넘으면 이탈 급증. 목적지·기간·일행만 받고 취향(Step 4)은 건너뛰기 가능.

### 3. 입력 방식 → **선택지 우선 + 자유 입력 보조**

한국 사용자는 키보드 입력 부담스러워함. 카드 탭이 가장 빠름.

### 4. 가치 약속 방식 → **헤드라인 + 미니 데모**

추상적 약속만 하지 않음. 실제 일정의 미니 버전을 첫 화면에 노출.

### 5. 다음 화면 연결 → **즉시 일정 생성**

마지막 단계 "일정 만들기" → 회원가입 없이 바로 생성 화면.

---

## 화면 구조

### Step 1 — 환영 + 가치 약속

- 헤드라인: "3분 안에 당신의 여행을 그려드려요"
- 부제: 우리 차별점 핵심 한 줄
- 미니 데모: 푸꾸옥 3박 4일 Day 1 샘플 일정 카드
- CTA: "시작하기" + "로그인 없이 바로 가능" 안내

### Step 2 — 목적지

- 검색 입력
- "한국인이 자주 가는 곳" 추천 4개 (푸꾸옥·다낭·도쿄·방콕)
- 각 카드에 "완전 지원" / "기본 지원" 라벨로 솔직한 한계 노출
- 푸꾸옥 기본 선택

### Step 3 — 기간·일행

- 출발일 (5월 14일 — 캘린더 모달은 추후)
- 박수 ± 스테퍼 (1~30)
- 동행 4가지: 혼자 / 친구·연인 / 가족 / 단체

### Step 4 — 취향 (건너뛰기 가능)

- 분위기 칩 (다중 선택, 1~3개): 맛집·관광지·사진명소·쇼핑·자연·현지핫플
- 페이스 칩 (단일): 여유롭게·균형있게·최대한 많이
- 제외할 것 (다중 선택): 알레르기 칩은 빨간 테두리
- "건너뛰기"와 "일정 만들기" 동시 노출

---

## 디자인 디테일

### 진행률 바

4칸으로 분할. 하나의 긴 바보다 "남은 단계가 명확"해짐.

### Step 1의 미니 데모

가치 약속을 추상에서 구체로. "이런 결과물을 받게 됩니다" 신호.
트리플과의 결정적 차별 — 트리플은 가입 전에 결과물 안 보여줌.

### Step 2의 "완전 지원" 라벨

다낭은 "기본 지원"이라고 미리 알려주는 게 신뢰 형성.
나중에 푸꾸옥 기능을 도쿄에서 못 찾았을 때 실망 방지.

### Step 4의 "건너뛰기"

같은 줄에 "건너뛰기"와 "일정 만들기"를 둠.
사용자에게 통제권을 줌. 강제가 아닌 선택.

## 인터랙션 패턴

| 동작 | 응답 |
|------|------|
| Step 진행 | 진행률 바 채워짐 + step 번호 변경 |
| 뒤로 가기 | 이전 단계로 + 진행률 바 줄어듦 |
| 카드 선택 | 보라/코랄 강조 + 2px 테두리 |
| 칩 토글 | 단일(페이스) vs 다중(분위기) 구분 |
| 마지막 CTA | 일정 생성 페이지로 navigation |

## 의도적으로 안 넣은 것

- ❌ 회원가입 강제 (트리플 실패 패턴)
- ❌ 이메일 입력 (일정 저장 시점에)
- ❌ 권한 요청 (위치·알림·카메라) (필요한 순간에 컨텍스트와 함께)
- ❌ 튜토리얼 슬라이드 4페이지 (사용자는 안 읽음)
- ❌ 소셜 로그인 버튼 (첫 화면 노출 부담)

## 추후 보완 사항

이 화면이 더 정교해질 때 추가할 것:

1. **캘린더 모달** (Step 3 출발일 탭 시) — 일자 선택 UI
2. **검색 결과 화면** (Step 2 검색창 입력 시) — autocomplete
3. **지원 도시 안내** (검색 결과에 없을 때) — "이 도시는 곧 지원 예정"
4. **취향 더 많은 옵션** — 동선 선호, 예산, 식이, 관심 카테고리
