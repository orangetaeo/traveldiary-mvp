## 답해야 할 단 하나의 질문

> **"AI가 진짜 일하고 있다는 신뢰의 시작점인가?"**

## 왜 이 화면이 LEVEL 1인가

10~20초 동안 무엇을 보여주느냐로 사용자의 AI 신뢰가 결정됩니다.
- **단순 "로딩..." 스피너** → 사용자 의심: "정말 일하는 거야?"
- **AI 사고 과정 단계별 시각화** → 사용자 신뢰: "와, 진짜 똑똑하네"

## 핵심 디자인 결정 (만들 때 정하기)

### 1. 단계별 시각화 vs. 진행률 바

**선택**: 둘 다. 큰 진행률 바 + 현재 단계 텍스트.

### 2. 어떤 단계를 보여줄까?

후보:
1. "당신의 취향 분석 중..."
2. "푸꾸옥의 식당 217곳 중 검토 중..."
3. "동선 최적화 계산 중..."
4. "5단계 환각 차단 검증..."
5. "일정 완성!"

각 단계 2~4초 = 총 12~20초.

### 3. 너무 빨리 끝나면?

LLM 응답이 5초만에 와도, 일부러 단계 보여주는 데 12초 사용.
사용자는 "뭐 이리 빨라?" 하면 오히려 의심함.
**페르세이브드 퀄리티 (perceived quality)** 원칙.

### 4. 너무 느리면?

20초 넘어가면 이탈 위험.
페일오버: 백그라운드에서 일정 생성, 30초 안에 안 오면 알림으로 전환.

### 5. 실패 시?

LLM 호출 실패, API 응답 없음 등.
- 명확한 에러 메시지: "잠시 문제가 있어요. 다시 시도할까요?"
- 폴백: 푸꾸옥 시드 데이터의 "기본 추천 일정" 즉시 제시.

## 인터랙션 패턴

| 시점 | 응답 |
|------|------|
| 페이지 진입 | 즉시 첫 단계 시작 + 백그라운드 LLM 호출 |
| 단계 완료 | 체크 마크로 변경 + 다음 단계 활성화 |
| 모든 완료 | 0.5초 "완성!" 표시 후 일정 화면으로 자동 navigation |
| 실패 | 친근한 에러 + "다시 시도" 또는 "기본 일정으로" |

## 데이터 흐름 (백엔드 — Phase 1에서 구현)

```typescript
// 1. 사용자 onboarding 데이터 받기
const tripDraft = {
  destination: "푸꾸옥",
  nights: 4,
  companion: "friends",
  preferences: { vibes, pace, excludes }
};

// 2. LLM 호출 (Claude 또는 GPT)
const aiItinerary = await generateItinerary(tripDraft);

// 3. 5단계 검증
const validatedItinerary = await validateItinerary(aiItinerary);
//    각 단계 결과를 SSE/WebSocket으로 클라이언트에 스트리밍
//    → 사용자에게 실시간 진행 상황 표시

// 4. DB 저장
const trip = await saveTrip(validatedItinerary);

// 5. 다음 페이지로 redirect
return router.push(`/itinerary/${trip.id}`);
```

## 의도적으로 안 넣을 것

- ❌ 진행률 숫자만 표시 (의미 없는 카운트다운)
- ❌ "잠시만 기다려주세요..." 같은 일반적 문구 (성의 없음)
- ❌ 광고 노출 (몇 초 시간이라도 로딩 시간에 광고는 신뢰 파괴)
- ❌ 회원가입 유도 (이 시점은 가치 발견 직전 — 방해 금지)
