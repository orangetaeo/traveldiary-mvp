# 02. Magic Moments

사용자가 "이 앱은 다르네"라고 느끼는 4개의 결정적 순간.

이 4개가 우리 앱의 **정체성을 드러내는 화면**입니다. MVP에서 이 4개만 환상적으로 만들면 사용자는 떠나지 않습니다.

---

## M1. 추천 근거 패널

**발생 시점**: 사용자가 일정 항목을 탭했을 때
**해당 화면**: 일정 상세 (`screens/04-itinerary-detail.md`)
**사용자 반응**: "어, 이거 트리플엔 없는데?" → 신뢰 형성

### 무엇을 보여주는가

AI가 추천한 모든 항목 옆에 **왜 골랐는지 근거**를 함께 표시.

예시 (푸꾸옥 즈엉동 야시장):
> **왜 이걸 골랐나**
> - 네이버 블로그 후기 412건 중 89% 긍정
> - 다음 일정(디아동 야경)과 도보 4분
> - 당신의 취향 "맛집·균형 페이스"와 일치

### 왜 이게 차별점인가

트리플·Layla·Mindtrip 모두 **결과만** 보여주고 근거는 안 보여줌.
근거 노출 = 사용자에게 통제권 = 락인.

### 구현 시 주의

- 근거는 진짜여야 함. "현지인 비율 60%" 같은 추정 불가능한 숫자 금지.
- 패널은 기본 접힘 상태. 사용자가 의심할 때 펼치게.
- 근거 부족할 땐 그냥 안 보여주는 게 나음.

## M2. D-Day 자동 모드 전환

**발생 시점**: 출발 당일 앱 첫 실행
**해당 화면**: 여행 중 홈 (`screens/05-travel-home.md`)
**사용자 반응**: "앱이 알아서 바뀌었네" → 락인

### 무엇을 바꾸는가

| 요소 | 여행 전 | 여행 중 |
|------|---------|---------|
| 헤더 | "푸꾸옥 3박 4일" | "DAY 2 · 13:42" |
| 통계 | 동선·경비 | 진행률 2/5 |
| 강조 색 | 보라 (계획) | 코랄 (진행) |
| FAB | 없음 | 카메라 번역 / 주변 검색 |

### 구현 시 핵심 룰

- 사용자는 모드 전환 버튼을 누른 적이 없어야 함.
- 같은 화면이 D-Day부터 자동으로 변신.
- 출발일 + 실제 위치(목적지 도시 안에 있음) 둘 다 충족 시 모드 전환.

## M3. Live Replan 첫 발동

**발생 시점**: 지각·날씨변동·웨이팅 발생
**해당 화면**: Live Replan (`screens/06-live-replan.md`)
**사용자 반응**: "이거 진짜 살아있네" → 입소문

### 어떻게 작동하는가

1. **트리거 감지**: 사용자 위치/시간이 일정과 ±15분 이상 벗어남
2. **3가지 옵션 제시**: 추천 / 안전 / 강행
3. **변경 영향 시각화**: 색상으로 즉시 파악 (초록=긍정, 앰버=부정, 검정=중립)
4. **사용자가 결정**: AI가 강제 안 함

### 핵심 룰

> **AI는 결정하지 않는다. 옵션을 제시하고, 사용자가 결정한다.**

이게 트리플·Layla와의 결정적 차이. 통제권을 사용자에게.

### 데이터 모델 — DAG

일정은 **리스트가 아닌 그래프(DAG)**로 설계.
- `flexibility`: fixed (고정) / flexible (유연) / booked (예약 완료)
- `priority`: 1~5 (5 = 절대 빼지 마)
- `flexMinutes`: ±N분 이동 가능

선형 리스트면 한 항목 바꾸면 도미노로 다 깨짐. 그래프여야 부분 재계산 가능.

## M4. 메뉴판 카메라 번역

**발생 시점**: 식당에서 메뉴판 촬영
**해당 화면**: 카메라 번역 (`screens/07-camera-translate.md`)
**사용자 반응**: 인스타 짤로 공유 → 바이럴

### 우리만의 접근

일반 번역기와 다른 점:

1. **한국인이 자주 시키는 메뉴를 위로**
   네이버 블로그 + 인스타 해시태그 분석으로 사회적 증거 노출.

2. **알레르기 필터 자동 적용**
   온보딩에서 입력한 알레르기를 매번 적용. 위험 메뉴는 빨간색.

3. **베트남어 원문 + 한국어 번역 함께**
   점원에게 가리키기 + 학습 효과. (Phase 2부터 일본어·태국어 추가)

### 바이럴 메커니즘

"친구에게 보내기" 하단 CTA → 카톡 공유 → 친구가 우리 앱 다운로드.
모든 사용자가 잠재적 마케터.

## 우선순위
