# Skill S-19: Librarian Search (도서관 검색)

> **스킬 유형**: 운영·품질
> **핵심**: INDEX → CATALOG → 책 펼치기 3단계 검색 패턴
> **사용 에이전트**: T19 Harness Librarian

## 검색 5단계

```
1. 키워드 추출         (사용자 요청 → 키워드 배열)
        ↓
2. INDEX.md 1차 검색   (목차에서 카테고리 매핑)
        ↓
3. CATALOG.md 2차 검색 (키워드 인덱스 + 상세 카탈로그)
        ↓
4. 개별 .md 펼치기      (최소 1~2개만)
        ↓
5. 팀 소집 명단 확정    (회의 멤버 + 참고 스킬)
```

## STEP 1. 키워드 추출

사용자 요청에서 핵심 명사/동사 추출:

```
"여행 중에 일정이 밀렸을 때 처리하는 거 만들어줘"
       ↓
키워드: [여행 중, 일정, 밀림(지연), 처리]
       ↓
도메인 매핑: M3 Live Replan
```

### 키워드 정규화

| 사용자 표현 | 정규 키워드 | 매핑 |
|------------|------------|------|
| "일정 짜기", "여행 만들기" | 일정 생성 | M1 |
| "왜 이거 추천했어?" | 근거 | M1, evidence |
| "일정 밀림", "지각" | replan | M3 |
| "메뉴 번역", "사진 번역" | OCR, 번역 | M4 |
| "여행 시작", "도착" | 모드 전환 | M2 |
| "DB 만들기" | Prisma, 스키마 | T14 |
| "배포" | Railway | T15 |

## STEP 2. INDEX.md 1차 검색

목차에서 다음을 확인:

1. 검색 패턴 표 (요청 유형 매핑)
2. 핵심 도메인 (M1~M4)
3. 인프라·품질 도메인 (T12~T19)
4. 절대 규칙 (3가지)

### 빠른 매핑 (INDEX의 검색 패턴 표)

| 요청 유형 | 매핑 |
|---------|------|
| "일정 만들어줘" | M1 + dag → T1, T2, T3, T4 |
| "여행 중 변경" | M3 + replan → T2, T5 |
| "메뉴 번역" | M4 + ocr → T6 |
| "기술 결정" | CTO + ADR → R1 + S-18 |
| "테스트" | QA + test → T12 + S-14 |
| "API 만들기" | API + audit → T10 + R4 + S-13 |
| "배포" | Railway + DevOps → T15 + S-10 |
| "보안" | security → T16 + S-11 |

## STEP 3. CATALOG.md 2차 검색

CATALOG에서 정밀 검색:

### 키워드 인덱스 활용

```
키워드 인덱스 표에서 한 번에 후보 확인.
예: "DAG, 그래프, 의존성" → T2, S-01
```

### 상세 카탈로그

각 후보의:
- 키워드 (재확인)
- 참조 스킬
- 연관 에이전트
- 언제 호출

→ 추가로 필요한 자료 발견 가능 (예: T4 호출 시 S-13도 함께)

## STEP 4. 책 펼치기 (최소 원칙)

> 너무 많이 열면 컨텍스트 낭비.
> **핵심 1~2개만** 우선 열고, 필요 시 추가.

### 우선순위

```
1. 메인 책임 에이전트 (1권)
2. 핵심 참조 스킬 (1권)
3. (필요 시) 협업 에이전트 또는 보조 스킬
```

### 펼치기 결정 기준

| 자료 | 펼치기? |
|------|---------|
| 메인 에이전트 .md | ✅ 항상 |
| 핵심 스킬 .md | ✅ 항상 |
| 협업 에이전트 .md | ⚠️ 회의 단계에서만 |
| 공유 라이브러리 | ⚠️ CTO 게이트 필요 시만 |
| 절대 규칙 스킬 (S-13, S-18) | ⚠️ 해당 트리거 발동 시만 |

## STEP 5. 팀 소집 명단

CATALOG의 "작업 유형별 책 묶음" 표를 기준으로:

```markdown
### 회의 멤버 (병렬 호출)
- R1 CTO (게이트)
- R2 PM (우선순위)
- T2 Itinerary Graph Engineer
- T5 Live Replan Engine
- T12 QA Lead
- T13 Code Reviewer

### 참고 자료 (펼침 완료)
- S-01 dag-scheduling
- S-06 live-replan-options
- S-13 audit-log-pattern (변경 API 있음)

### CTO 게이트?
- 알고리즘 변경 → S-18 적용

### 다음 단계
STEP 2 회의 진입 (위 멤버 병렬 호출)
```

## Triage 보고서 형식

검색 결과를 1단락의 Triage 보고서로:

```markdown
## Triage Report

**요청**: [한 줄 재정의]
**키워드**: [...]
**매직 모먼트**: M? (해당 시)

### 회의 멤버 (병렬 호출)
[...]

### 참고 자료
[...]

### CTO 게이트 필요?
[ Yes / No + 이유 ]

### 다음 단계
STEP 2 회의 진입
```

## 검색 누락 방지

### 자가 점검 4가지

```
□ 매직 모먼트 매핑됐는가?
□ CTO 게이트 트리거 확인했는가?
□ 감사 로그 트리거 확인했는가? (변경 API)
□ 다중 검증 4단계 멤버 모두 포함했는가?
```

### 누락 발견 시

회의 중 누락이 발견되면:
1. 즉시 멈춤
2. T19 재호출 → 추가 검색
3. CATALOG 키워드 인덱스 갱신 (다음에는 안 놓치도록)
