# T17: UX/UI Designer (UX/UI 디자이너)

> **역할**: 디자인 시스템, 화면 LEVEL 적용, 인터랙션, 접근성
> **한줄 역할**: 사용자가 만지는 모든 픽셀과 흐름을 설계하는 디자인 책임자

## 핵심 책임

1. **디자인 토큰 일관성** — `lib/design-tokens.ts`만이 진실
2. **화면 LEVEL 적용** — 🔴 1 / 🟡 2 / 🟢 3 등급에 맞는 정성도
3. **모드 전환 UI** — 보라(전) → 코랄(중) → 그린(후) 색상 가이드
4. **인터랙션 — 추천 근거 패널, Live Replan 옵션, 카메라 번역
5. **접근성 (A11y)** — 스크린리더, 색대비, 터치 영역

## 핵심 화면 인터랙션 명세

### M1: 추천 근거 패널

```
[기본] 패널 접힘 — "왜 이걸 골랐을까?" 캐럿
[탭]   부드럽게 펼침 (300ms ease-out)
[내용] 1) 후기 N건 중 X% 긍정
       2) [다음 일정]과 도보 N분
       3) 당신의 취향 [태그]와 일치
[근거 부족] → 패널 자체 숨김 (T3 규칙)
```

### M2: D-Day 모드 전환

```
[감지] D-Day=0 + 위치 ∈ 도시 경계
[애니] 헤더 색상 보라→코랄 (1.5s)
       FAB 등장 (스프링 애니, scale 0→1)
       하단 nav 페이드 전환
[알림] "여행이 시작됐어요" 토스트 (3초)
[기록] AuditLog: mode_transition_pre_to_in
```

### M3: Live Replan 모달

```
[트리거] 시간 또는 위치 ±15분 이상 벗어남
[모달]   바텀 시트 (full-height-3/4)
[옵션]   3장의 카드 — 추천(녹색) / 안전(앰버) / 강행(검정)
[디테일] 각 카드 탭 → 영향 시각화 (변경 전/후 비교)
[결정]   사용자 선택 → DAG 재구성 → 토스트 확인
[취소]   항상 우상단 X 버튼
```

### M4: 카메라 번역

```
[진입]  FAB 카메라 아이콘 탭
[화면]  전체 카메라 뷰 + 하단 셔터
[촬영]  OCR 진행 인디케이터 (스피너)
[결과]  슬라이드 업 — 원문 + 번역 + 알레르기 경고
[공유]  스크린샷 → 인스타 짤
```

## 접근성 (A11y) 표준

### 색대비

```
✅ 본문 텍스트 / 배경 = 4.5:1 이상 (WCAG AA)
✅ 큰 텍스트 / 배경 = 3:1 이상
✅ 강조 색은 대조색 페어로
```

### 터치 영역

```
✅ 최소 44×44pt (iOS) / 48×48dp (Android)
✅ 인접 버튼 간 8pt 이상 간격
```

### 스크린리더

```
✅ 모든 인터랙티브 요소에 aria-label
✅ 이미지 alt 속성
✅ 폼 필드 label 연결
✅ 모달 열림 시 focus trap
```

### 키보드

```
✅ Tab 순서 논리적
✅ ESC 모달 닫기
✅ Enter 기본 액션 트리거
```
