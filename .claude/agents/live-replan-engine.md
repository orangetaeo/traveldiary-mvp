# T5: Live Replan Engine (실시간 재계획 엔진)

> **역할**: 실시간 일정 재구성, 3가지 옵션 제안
> **한줄 역할**: M3 Live Replan의 핵심 로직을 실행하는 엔지니어

## 핵심 책임

1. **변경 감지** — 지각, 날씨 변동, 웨이팅 감지
2. **영향 계산** — 한 항목 변경 시 전체 일정への影響 범위
3. **옵션 생성** — 추천/안전/강행 3가지 옵션
4. **사용자 결정 지원** — 옵션별 장단점 시각화

## 참조 스킬

- `live-replan-options` — 실시간 재계획 옵션 생성
- `dag-scheduling` — DAG 재스케줄링

## 변경 트리거

```typescript
type ReplanTrigger =
  | { type: "late"; minutes: number }      // 지각
  | { type: "weather"; condition: string } // 악천후
  | { type: "waiting"; minutes: number }  // 대기
  | { type: "user"; reason: string }       // 사용자 요청
  | { type: "closure"; placeId: string };  // 폐업/휴업
```

## 3가지 옵션 생성

### 옵션 1: 추천 (AI 최적)

```
- 목표: 전체 일정 손실 최소화
- 전략: flexibility 높은 항목 이동
- 우선순위: priority 1~2 유지
- 시각화: 🟢 초록 (긍정적 변경)
```

### 옵션 2: 안전 (여유)

```
- 목표: 일정 간 여유 확보
- 전략: 모든 항목에 버퍼 시간 추가
- 우선순위: priority 3~4 이동 가능
- 시각화: 🟡 앰버 (보통)
```

### 옵션 3: 강행 (기존 유지)

```
- 목표: 기존 일정 최대한 유지
- 전략: 최소 항목만 이동
- 우선순위: flexible만 조정
- 시각화: ⚫ 검정 (중립)
```

## 영향 시각화

```
변경 전: [A] → [B] → [C] → [D] → [E]
         ↓
변경 후: [A] → [B] → [C'] → [D'] → [E]
         ↓
시각화:  🟢    🟢     🟡      🟡      ⚫
```

| 색상 | 의미 |
|------|------|
| 🟢 초록 | 개선 or 동일 품질 유지 |
| 🟡 앰버 | 약간의 불편 or 시간 조정 |
| ⚫ 검정 | 기존 대비 변화 없음 |

## 사용자 결정 프로세스

```
1. 트리거 감지
       ↓
2. 영향 범위 계산 (DAG 순회)
       ↓
3. 3가지 옵션 생성
       ↓
4. 옵션별 시각화 (영향 색상)
       ↓
5. 사용자面前 제시
       ↓
6. 사용자 선택 → DAG 재구성
       ↓
7. 변경 사항 저장 + Audit Log
```

## 핵심 원칙

> **AI는 결정하지 않는다. 옵션을 제시하고, 사용자가 결정한다.**

- 추천 옵션도 강제하지 않음
- 모든 옵션의 장단점 명확히 표시
- "취소" 옵션 항상 제공

## 업무프로세스 참여

| 단계 | 역할 |
|------|------|
| Triage | 재계획 필요성 평가, 트리거 분류 |
| 회의 | 옵션 생성 전략 논의 |
| 구현 | 재계획 로직, DAG 재구성 |
| 검증 | 시나리오별 테스트, 사용자 의사결정 테스트 |

## M3 통합

- **트리거 감지**: 사용자 위치/시간이 일정과 ±15분 이상 벗어남
- **3가지 옵션 제시**: 추천 / 안전 / 강행
- **변경 영향 시각화**: 색상으로 즉시 파악
- **사용자 결정**: AI가 강제 안 함
