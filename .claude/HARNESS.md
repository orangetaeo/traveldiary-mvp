# 🎮 하네스 운영 매뉴얼 — TravelDiary

> **하네스(HANES)** = "**H**armonized **A**gent **N**etwork & **E**volution **S**ystem".
>
> 사용자 요청이 들어오면 → 도서관 검색 → 에이전트 팀 병렬 소집 → 회의 → 구현 → **다중 검증** → 자가 진화.
>
> 한 번에 끝나는 작업은 없다. 여러 번 보고 다시 본다.

## 🚦 0. 절대 원칙

| 원칙 | 의미 |
|------|------|

   → T5 live-replan-engine.md 읽기

   → 회의: R1, R2, R3, T2, T5, T12, T13

## 🔬 5. 다중 검증 절차

### 4단계 검증 체크리스트

#### ① 자체 검증 (구현자)
- [ ] 타입 에러 없음 (`tsc --noEmit`)
- [ ] 디자인 토큰만 사용 (하드코딩 hex 없음)
- [ ] 기존 컴포넌트 재사용
- [ ] 명세 일치 (`docs/screens/XX.md`)
- [ ] 감사 로그 추가됨 (해당 시)

#### ② 코드 리뷰 (T13)
- [ ] 책임 단일성 (SRP)
- [ ] 가독성 (네이밍·구조)
- [ ] 에러 처리 적절성
- [ ] 테스트 가능성
- [ ] 보안 취약점 없음

#### ③ QA 테스트 (T12)
- [ ] 골든 패스 시나리오 통과
- [ ] 엣지 케이스 통과 (지연·중복·동시성)
- [ ] **환각 검출** — AI가 만든 데이터 진위 검증
- [ ] 한국어 특화 시나리오 (알레르기·후기)
- [ ] 모바일/PC 반응형

#### ④ CTO 사인오프 (R1)
- [ ] 아키텍처 일관성
- [ ] 보안/프라이버시 영향 점검
- [ ] 성능 영향 점검
- [ ] 기술 부채 평가
- [ ] ADR 필요 시 작성됨

> **하나라도 실패하면 STEP 3 (구현) 단계로 복귀.**
> "한 번 만들었으니 끝"이라는 평가는 절대 금지.

## 🧬 6. 자가 진화 루프

매 사이클 종료 시 T18 Self-Evolution Coach가:

```
1. 회고 (Retrospective)
   - 무엇이 잘 됐나 (Keep)
   - 무엇이 어려웠나 (Problem)
   - 다음에 무엇을 시도할까 (Try)

2. 학습 추출
   - 새 패턴 발견 → 스킬 신규 등록 또는 기존 갱신
   - 잘못된 가정 → memory에 anti-pattern 기록
   - 사용자 피드백 → memory/MEMORY.md 갱신

3. 도서관 갱신
   - INDEX.md 통계 업데이트
   - CATALOG.md 키워드 추가
   - 신규 .md 파일 추가 시 인덱싱

4. 메모리 저장
   - C:\Users\heros\.claude\projects\... 의 auto-memory에 저장
   - 다음 세션에서 자동 로드
```

자세한 절차: [skills/self-evolution-loop.md](skills/self-evolution-loop.md)
